Step 1 - Check Input Parameter Values for WinPE Kickstart

Connect via ssh:

ssh {automationworkerlinuxuser}@{automationworkerlinuxnode}

Execute the following script: 

                
if ! {isWinPEKickstart}
then
    echo "Please set isWinPEKickstart as the string 'true'"
    exit 1
fi

if [ {windowsFolderOnSamba} == "windows10" ]
then
    if ! {isWin10Bios} && ! {isWin10Uefi}
    then
        echo "Please set one of 'isWin10Bios' or 'isWin10Uefi' to true to install Win10."
        exit 1
    fi
fi

if [ {windowsFolderOnSamba} == "windows2016" ] ||
    [ {windowsFolderOnSamba} == "windows2019" ] ||
    [ {windowsFolderOnSamba} == "windows2022" ]
then
    if ! {isWinServerBios} && ! {isWinServerUefi}
    then
        echo "Please set one of 'isWinServerBios' or 'isWinServerUefi' to true to install Windows Server."
        exit 1
    fi
fi

Step 2 - Check Maximum Length of Win10 Hostname

Execute the following script: 

                
if {isWin10Bios} || {isWin10Uefi}
then

    HOSTNAME="{newOsNode.hostname}"
    
    LENGTH="${#HOSTNAME}"
    MAX_LENGTH="15"
    
    if [ "${LENGTH}" -gt "${MAX_LENGTH}" ]
    then
        echo "Length of hostname ${LENGTH} for Win10 can't be more than 15."
        false
    else
        echo "Length of hostname ${LENGTH} is less than or equal to 15."
    fi
    
else
    echo "Not Win10, skipping."
fi

Step 3 - Confirm Plain WinPE BIOS ISO Exists

Checks if the WinPE plain BIOS ISO {automationWorkerBaseDirectory}/winpe_plain_bios.iso exists.

If it does not, prints a message to ask user to run the WinPE BIOS ISO creation blueprint Create Windows PE (WinPE) Plain ISO on macOS or Linux Worker.

Only applies to BIOS kickstarts so if any of these parameters are set the string 'true':

  1. isWin10Bios
  2. isWinServerBios

Execute the following script: 

                
if {isWin10Bios} || {isWinServerBios}
then

    DIRECTORY="{automationWorkerBaseDirectory}"
    cd "${DIRECTORY}"
    
    ISO="winpe_plain_bios.iso"
    if [[ -f ${ISO} ]]
    then
        echo "${DIRECTORY}/${ISO} exists. Great."
    else
        echo "${DIRECTORY}/${ISO} does not exist. Please run blueprint \"Create Windows PE (WinPE) Plain ISO on macOS or Linux Worker\" to create it."
        exit 1
    fi
    
else
    echo "Skipping for UEFI ISO creation."
fi

Step 5 - Copy and Rename WinPE BIOS ISO

Makes a copy of the winpe_win2019_plain_bios.iso as winpe.iso in the folder {automationWorkerBaseDirectory}/build-{newOsNode.fqn.

This ISO is BIOS bootable and works for:

  1. Windows Desktop 10
  2. Windows Server 2016
  3. Windows Server 2019
  4. Windows Server 2022

Only applies to BIOS kickstarts so if any of these parameters are set the string 'true':

  1. isWin10Bios
  2. isWinServerBios

if {isWin10Bios} || {isWinServerBios}
then
    FOLDER="{automationWorkerBaseDirectory}/build-{newOsNode.fqn}"

    cd "${FOLDER}"
    cp --verbose "../winpe_plain_bios.iso" "winpe.iso"

Extract winpe.iso to a winpe_staging folder.

Step 8 - Deploy startnet.cmd

Deploys startnet.cmd.

This script is run by WinPE.

The startnet.cmd is run by WinPE. This script: 1. Installs drivers from "attune_drivers". 2. Sets the IP address. 3. Mounts the Samba server that hosts the extracted Windows ISOs as the "Z" drive. 4. Runs Windows setup.exe.

The Mako parameter "windowsFolderOnSamba" can have these values:

  • windows10
  • windows2016
  • windows2019
  • windows2022

Copy the File(s) WinPE_startnetcmd.tar to the target node.

{automationWorkerBaseDirectory}/build-{newOsNode.fqn}, relative to the home directory

Step 9 - Deploy Windows Unattended Config

Deploys the generic autounattend.xml.

Set one of these 'true' depending on whether you want Win10 or Windows Server for BIOS or UEFI boot methods.

  1. isWin10Bios
  2. isWin10Uefi
  3. isWinServerBios
  4. isWinServerUefi

Set isWinPEKickstart to 'true' for WinPE kickstarts.

For these Windows versions: * WIndows Desktop 10 * WIndows Server 2016 * WIndows Server 2019 * WIndows Server 2022

For these boot methods: * BIOS * UEFI

Copy the File(s) Windows_Unattended_config.tar to the target node.

{automationWorkerBaseDirectory}/build-{newOsNode.fqn}, relative to the home directory

Step 10 - Create Post Setup WIM

Creates a post_install.wim Windows Imaging Format file.

The purpose of this file is to overlay onto the C drive any files we want for the Windows Installer to use.

in this case we want to deploy a post install setup PowerShell script at C:\post_install_setup.ps1 that is called by autounattend.xml <FirstLogonCommands>'s section.

Step 10.1 - Deploy Post Setup PowerShell Script

Deploys the post install setup PowerShell script post_install_setup.ps1.

This file is called by the "" section in the autounattend.xml file.

This script is run once post installation of the WIndows operating system.

Copy the File(s) Post_Install_Setup_PowerShell_Script.tar to the target node.

{automationWorkerBaseDirectory}/build-{newOsNode.fqn}, relative to the home directory

Step 10.2 - Move Post Setup PowerShell Script to post_install_wim Folder

Moves post_install_setup.ps1 script to the post_install_wim folder.

Execute the following script:

Step 10.3 - Create post_install.wim Windows Image

Moves post_install_setup.ps1 into the folder post_install_wim.

The folder post_install_wim is the staging folder for creating the post_install.wim Windows Imaging Format file.

Execute the following script: 

                
DIR="{automationWorkerBaseDirectory}/build-{newOsNode.fqn}"
cd "${DIR}"

echo "We are in ${DIR}"

SOURCE="post_install_wim"
WIMFILE="post_install.wim"

echo "Creating ${WIMFILE} from directory ${SOURCE}."

wimcapture "${SOURCE}" "${WIMFILE}" --check

Step 11 - Modify WinPE boot.wim

Step 12 - Recreate BIOS Boot WinPE ISO from winpe_staging on macOS or Linux Worker

Creates a BIOS bootable ISO at {automationWorkerBaseDirectory}/kickstart_{newOsNode.fqn}.iso from the WinPE staging directory {automationWorkerBaseDirectory}/build-{newOsNode.fqn}/winpe_staging.

Only applies to BIOS kickstarts so if any of these parameters are set the string 'true':

  1. isWin10Bios
  2. isWinServerBios

Execute the following script: 

                
cd "{automationWorkerBaseDirectory}/build-{newOsNode.fqn}"

if {isWin10Bios} || {isWinServerBios} 
then
    ISO_NAME="kickstart_{newOsNode.fqn}.iso"
    
    if uname | \
        grep -iq "darwin"
    then
        echo "is macOS"
        export PATH="${HOME}/Parallels/homebrew/bin:$PATH"
        echo "Using $(which xorriso)"

        xorriso -as mkisofs \
            -b boot.img \
            -no-emul-boot \
            -c BOOT.CAT \
            -iso-level 3 \
            -udf \
            -joliet-long \
            -relaxed-filenames \
            -allow-limited-size \
            -J \
            -full-iso9660-filenames \
            -disable-deep-relocation \
            -omit-version-number \
            -V "KS_WIN" \
            -o "{automationWorkerBaseDirectory}/${ISO_NAME}" \
            winpe_staging

    else
        echo "not macOS"

        mkisofs \
            -b boot.img \
            -no-emul-boot \
            -c BOOT.CAT \
            -iso-level 3 \
            -udf \
            -joliet-long \
            -relaxed-filenames \
            -allow-limited-size \
            -J \
            -full-iso9660-filenames \
            -disable-deep-relocation \
            -omit-version-number \
            -V "KS_WIN" \
            -o "{automationWorkerBaseDirectory}/${ISO_NAME}" \
            winpe_staging
    fi

else
    echo "Skipping for UEFI ISO creation."
fi

Step 13 - Recreate UEFI Boot WinPE ISO from winpe_staging on macOS or Linux Worker

Execute the following script: 

                
if {isWin10Uefi} || {isWinServerUefi}
then
    cd "{automationWorkerBaseDirectory}/build-{newOsNode.fqn}"
    
    ISO_NAME="kickstart_{newOsNode.fqn}.iso"
    
    if uname | \
        grep -iq "darwin"
    then
        echo "is macOS"
        export PATH="${HOME}/Parallels/homebrew/bin:$PATH"
        echo "Using $(which xorriso)"
        
        xorriso -as mkisofs \
            -iso-level 4 \
            -full-iso9660-filenames \
            -rock \
            -b boot/etfsboot.com \
            -no-emul-boot \
            -boot-load-seg 0 \
            -boot-load-size 8 \
            -hide boot.catalog \
            -eltorito-alt-boot \
            -no-emul-boot \
            -b efi/microsoft/boot/efisys.bin \
            -boot-load-size 1 \
            -udf \
            -untranslated-filenames \
            -disable-deep-relocation \
            -o "{automationWorkerBaseDirectory}/${ISO_NAME}" \
            winpe_staging
            
    else
        echo "not macOS"

        mkisofs \
            -allow-limited-size \
            -iso-level 4 \ 
            -full-iso9660-filenames \
            -rock \
            -b boot/etfsboot.com \
            -no-emul-boot \
            -boot-load-seg 0 \
            -boot-load-size 8 \
            -hide boot.catalog \
            -eltorito-alt-boot \
            -no-emul-boot \
            -b efi/microsoft/boot/efisys.bin \
            -boot-load-size 1 \
            -udf \
            -untranslated-filenames \
            -disable-deep-relocation \
            -o "{automationWorkerBaseDirectory}/${ISO_NAME}" \
            winpe_staging
    fi
        
else
    echo "Skipping for BIOS boot."
fi
